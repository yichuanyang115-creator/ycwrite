# 调研指南（增强版）

## 关键词策略

### 核心词提取
从用户输入中提取核心关键词，保持原始表达不变。

### 细化词生成
在核心词基础上添加以下后缀：
- 教程
- 配置
- 实战
- 案例
- 原理
- 入门

### 禁止规则
**严禁**将关键词扩大到上位概念。

示例：
```
✅ 正确：
用户输入："Claude API"
核心词："Claude API"
细化词："Claude API 教程"、"Claude API 配置"

❌ 错误：
用户输入："Claude API"
扩展为："人工智能 API"、"大语言模型接口"
```

## 相关性打分规则

对每篇搜索结果进行打分：

```
基础分 = 0

+ 标题命中关键词：+3分
+ 摘要/正文命中关键词：+1分
+ 来自可信源列表：+2分
+ 官方文档：+2分
+ 发布时间在1个月内：+1分
```

### 时效性过滤

- **普通内容**：仅保留3个月内发布的内容
- **官方文档**：放宽至6个月内
- **用户提供的参考资料**：不受时效限制

### 最终筛选

保留10篇高质量内容：
- 核心词直接搜索：Top 5（按分数排序）
- 细化词扩展搜索：Top 3（按分数排序）
- 用户提供参考资料：全部保留（不计入10篇）

## 调研问题框架

针对每个主题，按以下5个维度展开调研：

### 1. 是什么（定义与概念）
- 该技术/概念的准确定义
- 与相关概念的区别和联系
- 关键术语解释

### 2. 如何工作（技术原理）
- 核心工作机制
- 技术架构或流程
- 关键算法或方法

### 3. 用在哪（应用场景）
- 当前主要落地场景
- 典型产品案例
- 行业应用现状

### 4. 未来趋势
- 近期重要进展（近6-12个月）
- 研究前沿方向
- 业界预测与判断

### 5. 争议与限制
- 已知局限性和问题
- 学界/业界争议点
- 伦理与安全挑战

## 搜索执行流程

### 步骤1：生成搜索关键词
```
核心词：{用户输入}
细化词组：
  - {核心词} + 教程
  - {核心词} + 配置
  - {核心词} + 实战
  - {核心词} + 案例
  - {核心词} + 原理
  - {核心词} + 入门
```

### 步骤2：并行搜索
同时执行3个信息源的搜索：
1. WebSearch：核心词 + 细化词（各取Top 4，共8条）
2. weixin_search_mcp：核心词（Top 5）
3. redbook_mcp：核心词（Top 5）

### 步骤3：打分与筛选
对所有结果（共18条）进行相关性打分，按分数排序后保留Top 10。

### 步骤4：深度提取
对Top 10文章使用WebFetch提取完整内容，按5个维度整理信息。

## 调研输出格式

### 1. 关键洞察（3-5条）
从所有来源中提炼的核心发现：
- 反常识观点或新趋势
- 数据亮点
- 行业共识

示例：
```
1. Claude API的上下文窗口已达200K tokens，是GPT-4的2倍
2. 90%的企业用户更关注成本而非性能极限
3. 官方推荐使用Prompt Caching可降低50%成本
```

### 2. 知识图谱
```
核心概念：{主题}
├── 子概念A
│   ├── 应用场景1
│   └── 应用场景2
├── 子概念B
│   ├── 技术原理1
│   └── 技术原理2
└── 相关技术
    ├── 竞品对比
    └── 生态工具
```

### 3. 源文章列表（Top 10）

| 标题 | 来源 | 发布时间 | 相关性分数 | 核心观点 |
|------|------|----------|-----------|---------|
| ... | ... | 2024-02-15 | 8分 | ... |

## 调研质量标准

- 至少包含1个优先级1或2的来源
- 数据/数字必须有来源支撑
- 不引用3个月以上的数据（官方文档6个月）
- 技术概念定义以官方文档为准
- 知识图谱层级不超过3层
- 关键洞察必须有数据或案例支撑
