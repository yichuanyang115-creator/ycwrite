# 大纲模板库

## 文章类型判断

根据主题自动选择最合适的模板：

| 主题类型 | 推荐模板 | 示例主题 |
|---------|---------|---------|
| 解释一个新技术/概念 | A：概念解释型 | "什么是RAG"、"Transformer架构解析" |
| 讲技术如何工作 | B：技术原理型 | "大模型是如何训练的"、"Diffusion模型原理" |
| 讲技术的商业/产品应用 | C：应用场景型 | "AI Agent在产品设计中的应用" |
| 评估/比较多个方案 | D：对比评测型 | "主流大模型横向对比" |
| 探讨争议性话题 | E：观点探讨型 | "AI会取代程序员吗" |

---

## 模板A：概念解释型

**适用**：向产品经理等非技术读者解释一个新技术或概念

```markdown
# 标题：[让普通人也能看懂的XXX]

## 引言（150-200字）
- 开场钩子：一个让读者感同身受的场景或问题
- 引出话题：为什么这个概念值得了解
- 预告收获：读完这篇文章你将了解什么

## 一、XXX到底是什么（300-500字）
- 用一句话给出最清晰的定义
- 用生活化类比帮助理解
- 澄清常见误解
[📷 配图：概念示意图/类比插图]

## 二、它是怎么工作的（400-600字）
- 拆解核心工作原理（步骤化）
- 每个步骤用具体例子说明
- 关键数据或技术参数
[📷 配图：工作流程图]

## 三、它能做什么（400-600字）
- 3-5个具体应用场景
- 每个场景配真实产品案例
- 强调产品经理视角的价值
[📷 配图：应用案例截图或示意图]

## 四、它的局限性是什么（300-400字）
- 2-3个主要限制
- 对应的解决思路或未来方向
- 帮助读者建立理性预期

## 结论（150-200字）
- 总结核心要点（3点以内）
- 给产品经理的行动建议
- 升华：对行业或工作的启示
```

## 模板B：技术原理型

**适用**：深入解析一个技术的工作机制，但保持科普可读性

```markdown
# 标题：[深入浅出：XXX的工作原理]

## 引言（150-200字）
- 提出一个令人好奇的问题
- 说明理解这个原理的价值
- 文章路线图

## 一、从直觉出发（250-350字）
- 用类比建立直觉认知
- 解释要解决的核心问题
[📷 配图：问题示意图]

## 二、核心机制拆解（600-800字）
- 分步骤讲解核心原理
- 每步骤配直观示例
- 关键公式/数据（简化呈现）
[📷 配图：核心机制流程图]

## 三、关键突破与创新（400-500字）
- 相比前代方法的改进点
- 解决了什么根本性问题
- 引用数据证明效果

## 四、实际运作中的挑战（300-400字）
- 理论与实践的差距
- 工程层面的关键挑战
[📷 配图：挑战示意图（可选）]

## 五、未来方向（200-300字）
- 当前研究热点
- 下一阶段可能的突破点

## 结论（150字）
- 核心原理总结
- 对产品人的实践意义
```

## 模板C：应用场景型

**适用**：面向产品经理，聚焦技术的实际商业和产品应用

```markdown
# 标题：[XXX如何重塑[行业/产品领域]]

## 引言（150-200字）
- 数据或案例开场（制造震撼感）
- 变化已经发生的证据
- 文章将覆盖的场景范围

## 一、技术背景速览（200-300字）
- 该技术是什么（一句话）
- 为什么现在爆发（关键时机）
[📷 配图：技术发展时间线]

## 二、核心应用场景（每场景300-400字，共3-4个场景）
每个场景包含：
- 场景描述与痛点
- 技术如何解决
- 真实产品案例
- 商业价值量化
[📷 每1-2个场景配一张配图]

## 三、落地挑战与关键成功因素（300-400字）
- 主要障碍（技术/组织/数据）
- 成功案例的共同特征
- 避坑建议

## 四、产品经理行动指南（300-400字）
- 如何评估是否适合引入
- 关键决策框架
- 与团队协作的建议

## 结论（150字）
- 趋势判断
- 最重要的一条建议
```

## 模板E：观点探讨型

```markdown
# 标题：[XXX？我们需要更清醒地看待这个问题]

## 引言（200字）
- 争议现象描述
- 两种极端观点
- 作者立场预告

## 一、支持者的论据（300-400字）
- 论据1 + 数据支撑
- 论据2 + 案例支撑
[📷 配图]

## 二、反对者的担忧（300-400字）
- 担忧1 + 依据
- 担忧2 + 依据

## 三、被忽视的关键变量（400-500字）
- 更全面的视角
- 真正决定结果的因素

## 四、理性判断框架（300字）
- 在什么条件下A成立
- 在什么条件下B成立

## 结论（150字）
- 作者综合判断
- 对读者的建议
```

## 大纲规范要求

1. **字数预估**：每个章节标注预计字数，总和与目标字数匹配（±10%）
2. **配图位置**：用 `[📷 配图：描述]` 明确标注，说明图片内容
3. **逻辑流**：章节之间有明确的递进或转折关系
4. **产品经理视角**：至少一个章节专门讨论对PM的价值/启示
