# Write Master — AI科普文章自动化写作

## 概览

一站式完成AI科普文章的创作全流程：**调研 → 大纲 → 写作 → 配图 → 排版输出**。

6个人工审核节点确保质量，Gemini API自动生成配图，最终输出带图的富文本HTML文件。

## 输入模式识别

开始前先判断用户的输入类型：

**极简模式**：只有主题关键词 → 执行完整调研，从阶段1开始
**指导模式**：提供了受众/参考资料/观点等 → 提取已有参数，补充缺失项
**续写模式**：提供了大纲或草稿 → 跳转至对应阶段

## 阶段1：需求确认与参数收集

### 步骤

1. 从用户输入中提取已有参数
2. 识别缺失参数，一次性询问（避免多轮追问）
3. 展示完整参数清单等待确认

### 参数清单

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 主题 | 必填 | 文章核心主题 |
| 目标受众 | 产品经理 | 可选：开发者/高管/普通大众/学生/创业者 |
| 文章长度 | 必须询问 | 见下方询问模板 |
| 核心观点 | 可选 | 文章主要论点 |
| 参考资料 | 可选 | URL或文档链接 |

### 文章长度询问模板

询问用户时，先给出推荐选项，再提供三个选择：

```
根据主题「{主题}」，我推荐【{推荐选项}】，原因是：{推荐理由}

请选择文章长度：
1. 短文（1000-2000字）— 快速概览，配图1张
2. 中篇（2000-5000字）— 深入浅出，配图2张  【AI推荐】
3. 长篇（5000字以上）— 全面解析，配图3张
```

### 配图数量规则
- 短文 → 1张
- 中篇 → 2张
- 长篇 → 3张

### ⏸️ 审核节点1：参数确认

展示所有收集到的参数，询问：「以上参数是否准确？是否需要调整？」

确认后输出参数配置，进入阶段2。

## 阶段2：主题调研（增强版）

### 关键词策略

从用户输入提取核心词，生成搜索关键词组：

**核心词**：保持用户原始输入不变
**细化词**：核心词 + [教程/配置/实战/案例/原理/入门]
**禁止**：扩大到上位概念（如"AI Agent"不扩展为"人工智能"）

示例：
- 用户输入："Claude API"
- 核心词："Claude API"
- 细化词："Claude API 教程"、"Claude API 配置"、"Claude API 实战"

### 多信息源并行搜索

同时从3个信息源获取内容：

| 信息源 | 工具 | 数量 | 时效性 |
|--------|------|------|--------|
| Web搜索 | WebSearch | Top 8 | 3个月内 |
| 微信公众号 | weixin_search_mcp | Top 5 | 3个月内 |
| 小红书 | redbook_mcp | Top 5 | 3个月内 |

**可信源列表**（额外加分）：
- 公众号：机器之心、量子位、新智元、AI前线、硅星人、极客公园、少数派、骆齐、hanniman
- 网站：sspai.com、36kr.com、jiqizhixin.com、juejin.cn、docs.anthropic.com、openai.com

### 筛选与打分规则

对每篇内容进行相关性打分：

```
基础分 = 0
+ 标题命中关键词：+3分
+ 摘要命中关键词：+1分
+ 来自可信源：+2分
+ 官方文档（时效放宽至6个月）：+2分
```

**时效性过滤**：
- 普通内容：仅保留3个月内
- 官方文档：放宽至6个月内

**最终保留**：10篇
- 核心词直接搜索：Top 5（按分数排序）
- 细化词扩展搜索：Top 3（按分数排序）
- 用户提供的参考资料：全部保留（不计入10篇限制）

### 调研输出

生成结构化调研报告，包含：

1. **关键洞察**（3-5条）
   - 从所有来源中提炼的核心发现
   - 反常识观点或新趋势
   - 数据亮点

2. **知识图谱**
   ```
   核心概念
   ├── 子概念A
   │   ├── 应用场景1
   │   └── 应用场景2
   ├── 子概念B
   └── 相关技术
   ```

3. **源文章列表**（按分数排序）
   | 标题 | 来源 | 发布时间 | 相关性分数 | 核心观点 |
   |------|------|----------|-----------|---------|

### ⏸️ 审核节点2：调研确认

展示：
- 关键洞察（3-5条）
- 知识图谱
- 源文章列表（Top 10）

询问：「调研方向是否正确？是否需要补充特定方面？」

## 阶段3：大纲生成（HOOK-BRIDGE-CORE-CLOSE结构）

### 文章结构框架

采用HOOK-BRIDGE-CORE-CLOSE四段式结构：

**HOOK（100-200字）** — 3秒抓住读者
- 反常识开头：颠覆认知的观点
- 场景代入：让读者身临其境
- 尖锐问题：直击痛点
- 故事开头：用故事引入

**BRIDGE（50-100字）** — 告诉读者能得到什么
- 明确价值主张
- 预告文章结构
- 建立阅读期待

**CORE（根据档位）** — 核心内容
每个章节包含：
- 概念/观点陈述（大白话）
- 类比/例子（让抽象变具体）
- 延伸思考
- 【可选】金句

**CLOSE（100-200字）** — 留下余韵
- 回扣开头
- 开放问题
- 视野拉远
- 个人感悟

### 步骤

1. 根据文章类型选择大纲模板（见 [references/outline-templates.md](references/outline-templates.md)）
2. 生成符合HOOK-BRIDGE-CORE-CLOSE结构的大纲
3. 在大纲中标注配图位置：`[📷 配图位置：建议图片内容描述]`
4. 标注字数预估（确保符合用户选择的档位）

### ⏸️ 审核节点3：大纲确认

展示完整大纲（含HOOK/BRIDGE/CORE/CLOSE标注、配图位置、字数预估）

询问：「大纲结构是否合理？HOOK是否足够吸引人？是否需要调整？」

## 阶段4：正文写作

### 禁用词列表

写作时严格避免以下表达：
- "众所周知"
- "让我们看看"
- "首先、其次、最后"（作为段落开头）
- "简单来说"

### 步骤

1. 读取 [references/writing-styles.md](references/writing-styles.md) 获取写作规范
2. 按HOOK-BRIDGE-CORE-CLOSE结构展开写作：

   **HOOK部分**：
   - 选择一种开头方式（反常识/场景/问题/故事）
   - 100-200字，3秒抓住注意力

   **BRIDGE部分**：
   - 50-100字，明确告诉读者能获得什么

   **CORE部分**：
   - 每个章节遵循：概念陈述 → 类比/例子 → 延伸思考
   - 应用写作原则：类比优先、数据支撑、产品视角、段落简短
   - 可选：在关键位置加入金句

   **CLOSE部分**：
   - 100-200字，留下余韵
   - 回扣开头或提出开放问题

3. 在章节对应位置保留配图占位符：`[IMAGE_PLACEHOLDER: 图片描述]`
4. 整合所有部分，确保过渡自然

### 前端实时同步

写作过程中，每完成一个部分（HOOK/BRIDGE/每个CORE章节/CLOSE），立即：
1. 将内容写入 `output/article_draft.md`
2. 更新进度状态到 `output/progress.json`：
   ```json
   {
     "stage": "writing",
     "completed_sections": ["HOOK", "BRIDGE", "CORE_1"],
     "current_section": "CORE_2",
     "progress_percent": 60,
     "last_update": "2024-03-03T21:45:00Z"
   }
   ```

前端可通过轮询 `progress.json` 实时显示写作进度。

### ⏸️ 审核节点4：初稿确认

展示完整文章初稿（Markdown格式，标注HOOK/BRIDGE/CORE/CLOSE各部分）

询问：「内容是否准确？HOOK是否吸引人？风格是否合适？是否需要修改特定段落？」

## 阶段5：配图生成

> **前提**：需要在项目中配置Gemini API key。
> 详见 [references/gemini-setup.md](references/gemini-setup.md)

### 步骤

1. 提取文章中所有 `[IMAGE_PLACEHOLDER: 描述]` 占位符
2. 为每张图读取 [references/image-prompts-guide.md](references/image-prompts-guide.md) 生成详细提示词
3. 运行 `scripts/gemini_image_gen.py` 批量生成图片：

```bash
python scripts/gemini_image_gen.py \
  --config config.json \
  --prompts prompts.json \
  --output ./output/images/
```

4. 验证图片生成成功，将占位符替换为实际图片路径

### API配置检查

运行前先检查config.json是否存在：

```bash
python scripts/config_loader.py --check
```

若配置缺失，提示用户参考 [references/gemini-setup.md](references/gemini-setup.md) 进行配置。

### ⏸️ 审核节点5：配图确认

展示所有生成的配图（以文件路径列表形式，并说明每张对应的文章位置）

询问：「配图是否合适？是否需要重新生成某些图片？」

---

## 阶段6：富文本排版与输出

### 步骤

1. 将配图占位符替换为实际图片路径
2. 运行 `scripts/markdown_to_html.py` 转换为富文本HTML：

```bash
python scripts/markdown_to_html.py \
  --input article.md \
  --images ./output/images/ \
  --template assets/html-template.html \
  --css assets/styles.css \
  --output ./output/article.html
```

3. 检查HTML格式、图片显示、样式应用
4. 生成最终交付物
5. 更新 `output/progress.json` 标记完成状态

### 前端部署准备

生成的HTML文件已包含：
- 响应式布局（移动端适配）
- 内联CSS样式
- Base64嵌入图片（无需额外图片文件）
- 可直接部署到任何静态托管服务

推荐部署平台：
- Vercel / Netlify（自动部署）
- GitHub Pages
- 阿里云OSS / 腾讯云COS

### ⏸️ 审核节点6：最终确认

告知用户输出文件路径，询问：「排版是否满意？是否需要调整样式？」
